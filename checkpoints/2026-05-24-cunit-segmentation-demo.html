C-Unit Segmentation Demo System

The demo uses Approach B with Approach C inside it: a structured local segmentation pipeline, plus rule-scoped rewrite agents that operate against machine-checkable artifacts. Official reference materials are used only to understand abstract formatting rules, never as fixtures, prompts, copied examples, or expected outputs.

Demo Contract

Inputs

  • Synthetic Descript-like transcript text.
  • Invented speakers and invented scene content.
  • Explicit forbidden-token guard list.

Gold Target

  • Synthetic segmented transcript target.
  • Rule coverage IDs stored with every case.
  • No official names, titles, or copied phrasing.

Evaluator

  • Deterministic score and failures.
  • Speaker, pause, timestamp, and notation metrics.
  • Official-source leakage guard.

Next System Boundary

The next funding-demo step is a local rewrite queue: each rewrite agent receives one case, one rule scope, the current draft, and the evaluator failures. The agent returns a new draft plus a JSON evidence packet. The verifier accepts or rejects the packet by rerunning the deterministic evaluator.
